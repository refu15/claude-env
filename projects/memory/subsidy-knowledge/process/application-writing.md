# 申請書ライティング

## 概要
補助金申請書の作成ノウハウをまとめたガイド。
書き方の基本構成から、採択率を上げるテクニックまでを記載する。

## 対象者
G2（申請書作成担当）、Admin

## 申請書の基本構成

補助金種別によって様式は異なるが、審査で評価される内容は共通している。

### 申請書の典型的な流れ
```
1. 事業者概要
   ├── 会社・事業の説明
   └── 代表者経歴

2. 現状と課題
   ├── 現状の事業状況
   ├── 直面している課題・問題
   └── 課題の背景・原因

3. 取組内容（補助事業の内容）
   ├── 具体的に何をするか
   ├── なぜこの取組を選んだか
   └── 取組のスケジュール

4. 期待される効果
   ├── 定量的な目標（数値で記載）
   ├── 事業者への効果
   └── 地域・社会への波及効果

5. 収支計画・費用明細
   └── 補助対象経費の内訳と根拠
```

## 書き方のポイント

### 原則1: 結論から書く
審査員は多数の申請書を読む。最初の2〜3文で「何をするか・なぜ重要か」が伝わるように書く。

**悪い例:**
> 「弊社は創業以来、〇〇業界で事業を続けてまいりました。近年の市場環境の変化により...（3行続く）...そのため今回、〇〇を導入することといたしました。」

**良い例:**
> 「人手不足による配送遅延を解消するため、自動仕分けシステムを導入する。現在、1日〇件の手動仕分けに〇時間を要しており、導入後は〇%の時間削減が見込める。」

### 原則2: 数字で語る
- 現状の課題: 「〇人で1日〇時間かけている」「売上が前年比〇%減少」
- 取組後の効果: 「〇時間削減（削減率〇%）」「新規顧客〇社獲得見込み」
- 市場規模: 「対象市場は〇億円規模で年率〇%成長」

### 原則3: 「なぜ自社か」を説明する
補助金は有限なため、審査員は「この事業者に補助する価値があるか」を見る。
自社の強み・経験・顧客基盤・技術力が取組と結びついていることを示す。

### 原則4: 審査基準に沿って書く
公募要領に審査基準が明記されている場合は、その基準に対応するように構成する。
「革新性」「実現可能性」「収益性」等の審査項目をチェックして、それぞれに対応する記述があるか確認する。

## フェルミ推定の活用

数値の根拠が薄い部分では、フェルミ推定を使って論理的に試算を作る。

### 活用場面
- 市場規模の推定（統計データがない場合）
- 導入効果（削減工数・売上増加）の試算
- 収支計画の売上予測

### 手順
1. 結論の数値を決める（例: 「年間売上1,000万円増加」）
2. 構成要素に分解する（例: 新規顧客数 × 客単価 × 購入頻度）
3. 各要素を説明可能な数値で埋める
4. 数式を申請書に記載して透明性を確保する

**例: 省力化による削減工数の試算**
```
現状: 手動作業 〇人 × 1日〇時間 × 営業日250日 = 年間〇時間
導入後: 自動化により〇%削減 → 年間〇時間削減
人件費換算: 〇時間 × 時給〇円 = 年間〇万円のコスト削減
```

### 注意点
- 推定根拠を申請書内に明示する（「〇〇の統計データをもとに試算」）
- 楽観的すぎる数値は審査員に疑問を持たれる。保守的な数値で説明できるように
- 実態からかけ離れた数値はNG（後の実績報告で達成できないとペナルティの可能性）

## グラフ・図の効果的な挿入ポイント

テキストだけでは伝わりにくい情報を視覚化することで審査員の理解を助ける。

### 挿入が効果的な箇所

| 挿入場所 | 推奨グラフ・図 | 目的 |
|---------|-------------|------|
| 現状の課題 | 折れ線グラフ（売上推移）| 課題の深刻さを視覚的に示す |
| 市場環境 | 棒グラフ（市場規模推移）| 市場の成長性を示す |
| 取組のフロー | フロー図 | 取組の全体像を一目で伝える |
| 導入前後の比較 | 表・Before/After図 | 効果を分かりやすく示す |
| 収支計画 | 折れ線グラフ（売上・費用・利益）| 収益化の見通しを示す |

### グラフ作成のルール
- タイトルを必ず付ける（「図1: 売上高の推移（〇〇〜〇〇年度）」）
- 出典を記載する（自社データ・業界統計・公的資料）
- 色は白黒印刷でも判別できるようにする（ハッチングを使う等）
- グラフの直後に「このグラフが示す通り〜」と文章で説明を加える

## 「通りやすい書き方」のパターン

### パターン1: 課題→取組→効果の三段構成
最もオーソドックスで審査員に読まれやすい構成。
「〇〇という課題があり（現状）、〇〇を行うことで（取組）、〇〇を達成する（効果）」という流れを一貫させる。

### パターン2: 競合・市場との差別化を強調
「競合他社は〇〇だが、当社は〇〇という強みを持つため、この取組で差別化できる」という構成。
特に新市場参入・新製品開発の申請で有効。

### パターン3: 顧客の声・ニーズを根拠にする
「既存顧客〇社からの要望を受けており、潜在需要が確認されている」という記述。
ヒアリングした実際の顧客の声（匿名可）を引用すると説得力が増す。

## 申請書作成の進め方

1. 情報整理（ヒアリングシートを元に要点をまとめる）
2. 骨子作成（見出しレベルで構成を固める。5〜10分で全体像を作る）
3. 初稿作成（箇条書きから文章に起こす）
4. 数値チェック（決算書・見積書と照合して数値の整合性を確認）
5. 事業者確認（内容の事実確認・修正依頼）
6. 最終仕上げ（誤字脱字・レイアウト整備）

## チェック項目（提出前の最終確認）
- [ ] 審査基準の各項目に対応する記述があるか
- [ ] 全ての数値に根拠があるか（フェルミ推定を含む）
- [ ] 申請書の数値と添付書類（決算書・見積書等）の数値が一致するか
- [ ] 補助対象外経費が含まれていないか
- [ ] ページ数・字数制限を超えていないか
- [ ] グラフ・図に出典が記載されているか
