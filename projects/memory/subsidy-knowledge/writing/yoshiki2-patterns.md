# 様式2（経営計画書兼補助事業計画書）構造パターン

最終更新: 2026-03-25
作成者: knowledge-builder
根拠: LEIMEI（広告代理店）、まりぼう（居酒屋）の採択事例分析

---

## 概要

小規模事業者持続化補助金の様式2は「経営計画（Table 7）」と「補助事業計画（Table 8）」の2部構成。
採択されるには、**課題→取組→効果**の一貫したストーリーが不可欠。

---

## Table 7: 経営計画書

### 全体の文字数目安

| 章 | 目安文字数 | 備考 |
|----|-----------|------|
| 1-1. 事業概要・沿革 | 200〜400字 | 創業ストーリーを含めると強い |
| 1-2. 売上・利益の状況 | 200〜300字 | 数値必須。3年分推移が理想 |
| 1-3. 経営課題 | 150〜250字 | 補助事業への橋渡しになる |
| 2-1. 市場の動向 | 300〜500字 | 全国→地域の2段階で記述 |
| 2-2. 顧客ニーズ | 200〜350字 | 実際の顧客の声があると説得力増 |
| 3. 強み・弱み | 200〜350字 | 番号付きで3〜4項目ずつ |
| 4-1. 経営方針・目標 | 200〜300字 | 定性＋定量の両方を書く |
| 4-2. 今後のプラン | 200〜350字 | 補助事業後のビジョンも含める |

---

### 1. 企業概要

#### 1-1. 事業概要・沿革

**書くべき内容:**
- 事業の一言説明（何を、誰に、どのように提供しているか）
- 創業年・代表者略歴（事業との関連性があれば特に有効）
- 事業規模（従業員数、所在地）
- 主要サービス・商品

**採択事例の書き方:**

まりぼう（物語型）:
> 「代表○○はバックパッカーとして東南アジアを旅した経験から、現地の家庭料理に魅了され、帰国後に料理を独学で習得。○年に天童市にて居酒屋まりぼうを開業。」

LEIMEI（機能型）:
> 「当社は○年創業のWeb専門広告代理店です。中小企業・スタートアップ向けに、SEO・SNS広告・リスティング広告の成果報酬型サービスを提供しています。従業員○名。」

**テンプレート:**
```
[代表者/創業者の経歴や事業開始の経緯 — 1〜2文]
[事業内容の説明 — 何を誰に提供しているか]
[現在の規模（従業員、拠点、主要顧客層）]
```

---

#### 1-2. 現在の売上・利益の状況

**書くべき内容:**
- 直近3年の売上推移（増減とその理由）
- 利益率・客単価（業種に応じて）
- 季節変動・特記事項

**数値記載ルール:**
- 売上は「○○万円」と具体的に
- 前年比を必ず計算して記載（例: 前年比16.8%増）
- コロナ等の外部要因があれば言及

**テンプレート:**
```
売上高は○年度○○万円、○年度○○万円（前年比○%増/減）、○年度○○万円（前年比○%増/減）と推移しています。
[増減の主な理由を1文]
客単価は○○円、月間顧客数は○○名程度です。
[現状における課題への橋渡し文]
```

---

#### 1-3. 経営課題

**書くべき内容:**
- 現状の売上・集客における具体的な問題
- その課題が生じている外部要因・内部要因
- 補助事業につながる課題（次章への伏線）

**重要:** この章で挙げた課題が、Table 8の補助事業で解決される構造にすること。

**テンプレート:**
```
現在の主な経営課題は以下の通りです。
① [課題1：集客・認知に関するもの]
② [課題2：売上・収益に関するもの]
③ [課題3：業務・設備に関するもの（任意）]
これらの課題解決のため、[補助事業の一言要約]に取り組む必要があります。
```

---

### 2. 市場の動向を踏まえた顧客ニーズ分析

#### 2-1. 市場の動向

**2段階構成（必須）:**

```
[全国・業界全体のマクロデータ]
  ↓
[地域・ローカルのミクロデータ]
```

**データの探し方:**
- 全国データ: 総務省・経済産業省・業界団体の統計（最新年のもの）
- 地域データ: 市区町村の統計、商工会議所データ、地域経済分析システム（RESAS）

**採択事例の書き方:**

LEIMEI:
> 「電通の調査によると、2024年の日本の総広告費は7兆6,730億円（前年比○%増）。うちインターネット広告費は○兆円と全体の○%を占め、年々拡大しています。一方、○○県内の中小企業においては、デジタル広告の活用率は○%にとどまっており...」

まりぼう:
> 「厚生労働省の調査では、飲食業の市場規模は○○兆円（2023年）と回復傾向にあります。天童市においても○○...」

**テンプレート:**
```
[業界・全国データ：出典付きで数値を1〜2個引用]
この傾向は当地域においても見られ、[地域データまたは地域の状況を説明]。
このことから、[事業に関連する機会・需要が高まっていると結論づける文]。
```

---

#### 2-2. 顧客ニーズ

**書くべき内容:**
- 主要顧客層のプロフィール（年齢、属性、行動パターン）
- 顧客が抱えている困りごと・ニーズ
- 実際の顧客の声（引用形式が最も説得力高い）
- 競合との比較における自社ポジション

**採択事例（まりぼう）の優れた書き方:**
> お客様より「場所が分かりづらくて迷った」「もっと広い席で食べたい」というご意見をいただいており...

**テンプレート:**
```
主要顧客層は[属性：年代・職業・地域など]で、[行動・購買パターン]という特徴があります。
顧客からは「[実際の声1]」「[実際の声2]」といったご要望をいただいています。
このことから、[顧客が本当に求めているもの]というニーズが高いと判断しています。
```

---

### 3. 自社の強み・弱み

**構成ルール:**
- 強み: 3〜4項目を番号付きで
- 弱み: 2〜3項目を番号付きで
- 弱みは補助事業で克服できるものを選ぶ

**強みの書き方（差別化が伝わる表現）:**

良い例:
> ① 一気通貫対応: 企画・制作・運用を社内完結するため、外注コストゼロ・スピード対応が可能
> ② SNS全般対応: Instagram・X・TikTok・YouTubeすべてに対応できる稀少な体制

悪い例（抽象的すぎる）:
> ① 高品質なサービス
> ② 豊富な経験

**テンプレート:**
```
【強み】
① [具体的な強み1：何が、なぜ強みなのかを説明]
② [具体的な強み2]
③ [具体的な強み3]
④ [具体的な強み4（任意）]

【弱み】
① [弱み1：補助事業で克服できるもの]
② [弱み2]
③ [弱み3（任意）]
```

---

### 4. 経営方針・目標と今後のプラン

#### 4-1. 経営方針・目標

**必須要素:**
- 定性目標（どんな会社・事業にしたいか）
- 定量目標（売上・客数・客単価など数値で）
- 目標年度（補助事業完了後1〜3年）

**テンプレート:**
```
【定性目標】
[事業のビジョン・目指す姿を1〜2文で]

【定量目標（補助事業完了後○年以内）】
- 売上高: ○○万円（現状比○%増）
- 客数/顧客数: 月○○名（現状比○%増）
- 客単価: ○○円
```

---

#### 4-2. 今後のプラン

**書くべき内容:**
- 補助事業実施後の展開
- 中期的な事業拡大計画
- 補助事業との連動性

---

## Table 8: 補助事業計画書

### 1. 補助事業で行う事業名（30文字以内）

**ルール:**
- 30文字以内（厳守）
- 「何をするか」が一目で分かる具体的な名称
- 抽象的な言葉（「業務改善」「集客強化」）は避ける

**採択事例:**
- 「ホームページリニューアルおよび営業資料作成による新規顧客開拓」→ 長すぎる場合は短縮
- 「天童市繁華街への移転・内装工事による新規顧客獲得」

**テンプレート:**
```
[具体的な取組内容]による[期待される成果]
```

---

### 2. 販路開拓等の取組内容

#### 事業の概要

**1〜3文で全体像を説明。**

テンプレート:
```
本補助事業では、[具体的な取組]を実施します。
[なぜその取組を行うのか — 経営課題との接続]
これにより、[期待される効果]を目指します。
```

---

#### 背景・目的

**経営計画で述べた課題と直結させる。**

チェックポイント:
- Table 7の「1-3. 経営課題」と整合しているか
- 「なぜ今この事業が必要か」が伝わるか

---

#### 具体的な取組

**最も詳細に書く章。以下を含める:**
- 何を（具体的な発注内容・仕様）
- 誰と（業者・パートナー）
- どのように（手順・プロセス）
- どの程度（数量・規模）

**採択事例（まりぼう）の優れた記載:**
> 「移転先は天童市○○町○丁目、現在より○m²広い25席規模の物件。工事はA社（地元施工業者）に依頼し、多国籍料理の世界観を表現したインテリアデザインを採用する。工事期間中はInstagramで進捗を発信し、オープン前から認知度向上を図る。」

---

#### 事業実施スケジュール

**月単位で記載（必須）。**

| 時期 | 取組内容 |
|------|---------|
| ○月 | 業者選定・契約 |
| ○月〜○月 | 制作・工事実施 |
| ○月 | 完成・公開・オープン |
| ○月以降 | 運用・効果測定 |

---

### 3. 業務効率化の取組内容（任意）

**記載する場合:**
- 上限補助額が増える可能性あり
- 販路開拓との連動性を説明する

---

### 4. 補助事業の効果

#### 定性効果

**書き方:**
- 数値で表せない価値（ブランド向上・顧客満足度など）
- 地域・社会への貢献

テンプレート:
```
① [ブランド・認知面の効果]
② [顧客体験・サービス向上の効果]
③ [地域・社会への波及効果（任意）]
```

---

#### 定量効果（売上目標・数値シミュレーション）

**フェルミ推定形式で記載（採択率が高い書き方）。**

基本構造:
```
1日あたり売上 = 客数 × 客単価
            = ○○人 × ○,○○○円
            = ○○,○○○円

年間売上 = 1日あたり売上 × 営業日数
        = ○○,○○○円 × ○○○日
        = ○,○○○万円
```

採択事例（まりぼう）:
> 「移転後の想定客数は1日11.3人（現在比○%増）、客単価4,500円（現在4,000円）とすると、1日売上50,850円、年間1,330万円を見込みます。」

## 採択事例から学ぶ章立てチェックリスト

### Table 7 チェックリスト

- [ ] 1-1: 創業ストーリーまたは事業説明が具体的に書かれているか
- [ ] 1-2: 売上推移が数値（万円・前年比）で記載されているか
- [ ] 1-3: 経営課題がTable 8の補助事業と対応しているか
- [ ] 2-1: 全国データ + 地域データの2段階で市場を説明しているか
- [ ] 2-1: データに出典（○○省、○○調査など）が付いているか
- [ ] 2-2: 顧客の声（実際の言葉）が引用されているか
- [ ] 3: 強みが具体的で差別化が伝わるか（抽象的な表現になっていないか）
- [ ] 3: 弱みが補助事業で克服できるものになっているか
- [ ] 4-1: 定性目標と定量目標の両方があるか
- [ ] 4-2: 補助事業完了後の展開まで書かれているか

### Table 8 チェックリスト

- [ ] 事業名が30文字以内か
- [ ] 事業名を読んだだけで何をするか分かるか
- [ ] 背景・目的がTable 7の課題と整合しているか
- [ ] 具体的な取組に「何を・誰と・どのように・どの程度」が含まれているか
- [ ] スケジュールが月単位で記載されているか
- [ ] 定性効果と定量効果が分けて書かれているか
- [ ] 定量効果がフェルミ推定形式で計算されているか
